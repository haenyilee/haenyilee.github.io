# 서브쿼리

## 서브쿼리가 생겨난 이유
- 여러 쿼리문장을 합쳐서 한 번에 사용할 수 있게 하기 위해서이다.
- 속도가 빠르다는 장점이 있다.
  - 여러 쿼리 문장을 따로 실행하면 연결/해제를 반복해야 하기 때문에 느릴 수밖에 없다.

## 서브쿼리의 종류

- 단일행
  - 출력 결과가 하나인 것
  - `WHERE sal = (SELECT AVG(sal) ~~ )`
  
- 다중행
  - 출력 결과가 여러개인 것
  - `deptno IN (SELECT deptno FROM dept)`
  
- 다중컬럼

- 스칼라 서브쿼리
  - 실무에서 가장 많이 사용됨
  - 컬럼 대신 사용될 수 있다.
  - SELECT no, (SELECT ~) ...
  - JOIN을 걸지 않아도 되어서 편리하다.
